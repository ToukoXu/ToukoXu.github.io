Touko2022-02-012022-02-01

一、事件处理

1. 事件流三阶段

  • 捕获阶段 (Capture Phase)：事件从 window 向下传播到目标元素
  • 目标阶段 (Target Phase)：事件到达目标元素本身
  • 冒泡阶段 (Bubble Phase)：事件从目标元素向上冒泡到 window

2. 事件监听方式对比

方式 示例 优点 缺点
HTML 属性 <button onclick="handle()"> 简单直接 混合 HTML/JS，不易维护
DOM 属性 btn.onclick = handle 简单 只能绑定一个处理函数
addEventListener btn.addEventListener('click', handle) 推荐方式，支持多个监听器 需要手动移除

3. addEventListener()

语法

参数

  • useCapture（旧版本）

    类型 默认值 作用
    Boolean false true：沿着 DOM 树向上冒泡的事件不会触发监听器
    和上面的{ capture: true / false }在捕获功能上完全等效

  • options （Object）

    选项 类型 默认值 作用
    capture Boolean false 决定监听器在事件流的哪个阶段被触发
    false：冒泡阶段，true：捕获阶段
    once Boolean false true：最多调用一次监听器，调用后自动移除
    passive Boolean false true：监听器不会调用preventDefault()
    signal AbortSignal（信号对象） 调用abort()时，移除监听器

1. Event（基础事件）

所有事件的基类，提供基本的事件属性和方法

构造函数

属性：

  • type：事件类型（如 “click”）
  • bubbles：是否冒泡
  • cancelable：是否可以取消默认行为
  • target：事件目标，可以用来实现事件委托
  • currentTarget：始终为绑定事件的元素
  • eventPhase：事件当前阶段（0=NONE, 1=CAPTURING, 2=AT_TARGET, 3=BUBBLING）
  • timeStamp：事件发生的时间戳
  • isTrusted：事件是否由用户触发（true）还是脚本创建（false）
  • defaultPrevented：是否已阻止默认行为
  • composed：事件是否可以穿过 Shadow DOM 边界

方法：

  • preventDefault()：阻止默认行为，可以使用Event.cancelable来检查该事件是否支持取消
  • stopPropagation()：阻止事件传播
  • stopImmediatePropagation()：阻止其他监听器执行
  • composedPath()：返回事件路径数组

具体事件类型：

事件 触发时机
select 选中文本
submit 表单提交
reset 表单重置
change 值改变并失去焦点
load 资源加载完成
unload 离开页面
DOMContentLoaded DOM 解析完成
resize 窗口大小改变
scroll 滚动元素
hashchange URL 哈希改变
play 启动媒体
pause 暂停媒体

2. UIEvent（用户界面事件）

继承自Event，处理与浏览器UI相关的事件

特有属性：

  • view：关联的窗口对象
  • detail：事件详情（如点击次数）

2.1 MouseEvent（鼠标事件）

继承自UIEvent，处理所有鼠标相关事件

特有属性：

  • screenX, screenY：屏幕坐标
  • clientX, clientY：视口坐标
  • pageX, pageY：文档坐标
  • screenX, screenY：屏幕坐标
  • offsetX, offsetY：目标元素坐标
  • button：按下的鼠标按钮（0=左键, 1=中键, 2=右键）
  • buttons：按下的多个按钮
  • relatedTarget：相关元素（如 mouseover 时的来源元素）

具体事件类型：

事件类型 触发时机 是否冒泡
click, dblclick 点击（按下并释放）, 双击 是
mousedown, mouseup 鼠标按下, 鼠标释放 是
mousemove 鼠标移动 是
mouseenter, mouseleave 鼠标进入元素, 鼠标离开元素 否
mouseover, mouseout 鼠标进入元素或子元素, 鼠标离开元素或子元素 是
contextmenu 右键菜单 是

2.1.1 WheelEvent（滚轮事件）

继承自MouseEvent，处理滚轮/触控板滚动事件

特有属性：

  • deltaX：水平滚动量（像素）
  • deltaY：垂直滚动量（像素）
  • deltaZ：Z轴滚动量（3D设备）
  • deltaMode：滚动单位（0=像素, 1=行, 2=页）

具体事件类型：

事件类型 触发时机
wheel 滚轮滚动

2.1.2 DragEvent（拖放事件）

继承自MouseEvent，处理拖放事件

特有属性：

  • dataTransfer：在拖放交互期间传输的数据

具体事件类型：

事件类型 触发时机
drag 拖动元素（含选择的文本，下同）
dragstart, dragend 开始拖动元素, 拖动操作结束（释放鼠标按钮或按下退出键）
dragover 元素拖动到有效放置目标上（每几百毫秒）
dragenter, dragleave 拖动的元素进入放置目标, 拖动的元素离开放置目标
drop 在放置目标上放置元素

2.1.3 PointerEvent（指针事件）

继承自MouseEvent，统一鼠标、触摸、触控笔事件

特有属性：

  • pointerId：唯一指针ID
  • width, height：接触区域尺寸
  • pressure：压力值（0-1）
  • pointerType：设备类型（”mouse”, “pen”, “touch”）
  • isPrimary：是否为主指针

具体事件类型：

事件类型 触发时机
pointerdown, pointerup 指针按下, 指针释放
pointermove 指针移动
pointerover, pointerout 指针进入元素, 指针离开元素
pointerenter, pointerleave 指针进入元素, 指针离开元素
pointercancel 指针中断
gotpointercapture, lostpointercapture 元素启用捕获后触发, 捕获被释放后触发

2.2 KeyboardEvent（键盘事件）

继承自UIEvent，处理键盘输入事件

特有属性：

  • key：按键的字符串表示（如 “Enter”）
  • code：物理按键代码（如 “KeyA”）
  • location：按键位置（0=标准, 1=左侧, 2=右侧, 3=数字键盘）
  • repeat：是否重复按键
  • isComposing：是否在输入法编辑中
  • 修饰键状态：altKey, ctrlKey, shiftKey, metaKey

方法：

  • getModifierState(key)：检查特定修饰键状态

具体事件类型：

事件类型 触发时机 是否冒泡
keydown, keyup 按下任意键, 释放按键 是

2.3 FocusEvent（焦点事件）

继承自UIEvent，处理元素焦点变化

特有属性：

  • relatedTarget：相关元素（如失去焦点时获得焦点的元素）

具体事件类型：

事件类型 触发时机 是否冒泡 替代方案
focus, blur 元素获得焦点, 元素失去焦点 否 focusin, focusout
focusin, focusout 元素获得焦点, 元素失去焦点 是 -

2.4 TouchEvent（触摸事件）

继承自UIEvent，处理触摸屏设备交互

特有属性：

  • touches：当前所有触摸点
  • targetTouches：当前元素上的触摸点
  • changedTouches：本次事件相关的触摸点

具体事件类型：

事件类型 触发时机
touchstart 触摸开始
touchmove 触摸移动
touchend 触摸结束
touchcancel 触摸中断

2.5 InputEvent（输入事件）

继承自UIEvent，处理用户输入事件

特有属性：

  • data：插入的字符
  • dataTransfer：拖放或插入/删除的数据
  • inputType：更改的类型（inserting或deleting）

具体事件类型：

事件类型 触发时机
input 用户输入

3. AnimationEvent（动画事件）

继承自Event，处理CSS动画相关事件

特有属性：

  • animationName：动画名称
  • elapsedTime：动画已运行时间（秒）
  • pseudoElement：关联的伪元素（如 “::before”）

具体事件类型：

事件类型 触发时机
animationstart, animationend 动画开始, 动画结束
animationiteration 动画重复播放
animationcancel 动画被取消

4. TransitionEvent（过渡事件）

继承自Event，处理CSS过渡效果相关事件

特有属性：

  • propertyName：发生过渡的CSS属性
  • elapsedTime：过渡已运行时间（秒）
  • pseudoElement：关联的伪元素

具体事件类型：

事件类型 触发时机
transitionrun 过渡创建时
transitionstart 过渡实际开始时
transitionend 过渡完成时
transitioncancel 过渡被取消时

5. ClipboardEvent（剪贴板事件）

继承自Event，用于处理剪切板相关信息的事件

特有属性：

  • clipboardData：受剪贴板操作影响的数据

具体事件类型：

事件类型 触发时机
copy 复制元素的内容
cut 剪切元素的内容
paste 将内容粘贴到元素中

6. CustomEvent（自定义事件）

继承自Event，用于创建自定义事件

特有属性：

  • detail：自定义数据

方法：

  • initCustomEvent()：初始化自定义事件

三、性能优化

1. 高频事件的防抖（debounce）与节流（throttle）

2. 使用事件委托（减少DOM操作，且支持动态内容）

3. 一次性事件（{ once: true }）

4. 被动事件监听器（{ passive: true }）

5. 事件顺序控制（{ capture: true }）

具体应用案例

  • 全局快捷键
  • 关闭下拉菜单/弹窗
  • 表单自动保存
  • 事件拦截，如：禁用页面某些区域的点击

6. 事件移除（removeEventListener）

7. 事件批量移除（{ signal: controller.signal }）

8. 事件选择原则

  • 优先使用不会冒泡的事件：如 mouseenter/mouseleave 替代 mouseover/mouseout
  • 移动端优先指针事件：pointerdown 替代 mousedown/touchstart
  • 焦点事件使用冒泡版本：focusin/focusout 替代 focus/blur

四、常见问题与解决方案

1. scroll事件触发频繁导致卡顿

2. 移动端点击延迟300ms

3. 事件内存泄漏

4. 多个事件监听器执行顺序不可控

5. 浏览器兼容性问题

6. 事件总线模式
